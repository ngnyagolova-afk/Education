Кули на Ханой — рекурсивно решение
Рекурсивна идея
•  Задача: премести n диска от A на C, използвайки B
•  Стъпка 1: премести (n-1) диска от A на B (използвайки C)
•  Стъпка 2: премести най-долния диск от A на C
•  Стъпка 3: премести (n-1) диска от B на C (използвайки A)
•  Базов случай: n == 1 → директно преместване A → C
•  Брой ходове: 2ⁿ − 1
static void Hanoi(int n, char a, char b, char c)
{
if (n < 2)
Console.WriteLine(a.ToString() + "->" + c.ToString());
else
{
Hanoi(n - 1, a, c, b);
Console.WriteLine(a.ToString() + "->" + c.ToString());
Hanoi(n - 1, b, a, c);
}
}
static void Main(string[] args)
{
int n = Convert.ToInt32(Console.ReadLine());
Hanoi(n, '1', '2', '3');
}
При n=3: нужни са 2³−1 = 7 хода    |    При n=10: 1023 хода    |    При n=64: ~1.8×10¹⁹ хода

Свойства
•  Cₙᵏ = n! / (k! · (n−k)!)
•  Всеки елемент е сума от двата над него
•  C(n,k) = C(n-1,k-1) + C(n-1,k)
•  Лява и дясна страна — единици
•  Симетрия: C(n,k) = C(n, n-k)
•  n-тия ред = коефициенти на (a+b)ⁿ

Триъгълник на Паскал — C# реализация
Вместо рекурсивно изчисление на n!, използваме програмна реализация на Паскал:
static int N = 16;
static long[] bas = new long[100];
static long[] now = new long[100];
static void Main(string[] args)
{
bas[0] = 1;
for (int t = 1; t <= N; t++)
{
for (int i = 0; i < t + 1; i++)
now[i] = 0;
for (int i = 0; i < t; i++)
{
now[i]     += bas[i];   // от ляво
now[i + 1] += bas[i];   // от дясно
}
for (int i = 0; i <= t; i++)
bas[i] = now[i];
}
// bas[] съдържа n-тия ред на триъгълника на Паскал
Console.WriteLine("C(N,k) за N=" + N + ":");
for (int k = 0; k <= N; k++)
Console.Write(bas[k] + " ");
}
Защо Паскал?
•  n! много бързо преполнява int/long
•  Паскал: само събиране — без преполване
•  C(16,8) = 12870 — лесно се изчислява
•  Използва два реда масив: bas[] и now[]
•  Работи до N≈66 за long (64-битов)
•  Крайните числа в реда → C(N,k) за всяко k

4×4 дъска — едно решение
Условие на задачата
•  Поставете N дами на N×N дъска
•  Никоя дама да не заплашва друга
•  Дама заплашва: ред, колона и диагонали
•  При N=8: 92 решения
•  При N=4: 2 решения
•  Алгоритъм: backtracking (с връщане)
Проверки
•  Ред: col[row] — само 1 дама на ред
•  Колона: colUsed[] — само 1 дама на колона
•  Главен диагонал: row - col = const
•  Странич. диагонал: row + col = const
•  diagUsed1[row-col+n]
•  diagUsed2[row+col]
Поставяме по 1 дама на ред, рекурсивно преминаваме към следващия ред.

Задача с дамите — C# реализация
static int n;
static int[]  col       = new int[100];
static bool[] colUsed   = new bool[100];
static bool[] diagUsed1 = new bool[200]; // row - col
static bool[] diagUsed2 = new bool[200]; // row + col
static int count = 0;
static void Place(int row)
{
if (row >= n) { count++; Print(); return; }
for (int t = 0; t < n; t++)
{
if (!colUsed[t] && !diagUsed1[row - t + n] && !diagUsed2[row + t])
{
col[row] = t;
colUsed[t]         = true;
diagUsed1[row-t+n] = true;
diagUsed2[row+t]   = true;
Place(row + 1);
colUsed[t]         = false;
diagUsed1[row-t+n] = false;
diagUsed2[row+t]   = false;
}
}
}
// В Main: n=int.Parse(Console.ReadLine()); Place(0);
Как работи
•  Поставяме 1 дама на всеки ред
•  Проверяваме колона и двата диагонала
•  При успех: рекурсия за следващия ред
•  При неуспех: пробваме следваща колона
•  Backtracking: връщаме маркировките
•  Базов случай: row == n → решение!
•  Сложност: O(n!) в най-лош случай
•  N=8 → 92, N=12 → 14200 решения

Цветни комбинации — задача
Задача: Имаме квадрат 3×3, в него всяко квадратче може да се оцвети в един от 9 цвята. На всяка секунда трябва да се сменят четири ъглови цвята в произволен ред. Скидулирайте графично приложение ШАДРАВАН, визуализиращо смяната.
3×3 квадратна мрежа
(9 различни цвята)
Правила на играта
•  9 квадратчета, всяко с различен цвят
•  На всяка секунда: смяна на 4 ъглови цвята
•  Визуализация: WinForms + Timer
•  Генерираме комбинации от 9 по 4
•  Всяка комбинация = различна смяна
•  Показваме текущата комбинация
Математика
•  Брой комбинации: C(9,4) = 126
•  Редът не е важен → комбинации
•  На всяка 1 секунда: следваща
•  Лексикографски ред на смените
•  Изчерпваме всички 126 смени
•  Визуализация на всяка стъпка
